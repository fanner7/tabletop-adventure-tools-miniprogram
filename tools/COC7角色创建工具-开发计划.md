## ⚠️ 重要设计原则（手机端）

### 1. 本地存储，无云端
- 微信小程序没有云端服务器
- 所有角色数据通过 `wx.setStorageSync()` / `wx.getStorageSync()` 保存在手机本地
- 角色列表存储在本地，导出分享通过截图或复制文本

### 2. 手机操作友好
- **大按钮、大点击区域**：所有可点击元素 ≥ 44pt（iOS HIG 规范）
- **单手操作**：核心按钮放屏幕下半部分
- **滚动优先**：避免横向滚动，内容纵向排列
- **即时反馈**：掷骰动画、选中高亮、轻触震动反馈
- **输入降噪**：尽可能用选择器（picker）代替键盘输入；必须输入时弹出数字键盘
- **防误触**：重要操作（如清空角色）需二次确认弹窗

### 3. 性能
- 技能数据（30+ 职业 × 50+ 技能）启动时一次性加载到内存
- 避免频繁 `setData` 大对象，采用差分更新
- 武器库/防具库等大数据用懒加载

## 🟢 第一阶段：核心建卡流程（MVP）✅ 已完成

> **目标：能跑通完整的建卡流程，生成一张可用的角色卡**

### 1.1 页面结构与步骤引导
```
┌─────────────────────────┐
│  步骤指示器 (Step 1/5)   │  ← 顶部圆点进度条
├─────────────────────────┤
│                         │
│   当前步骤内容区域         │  ← 可滚动
│   （属性/信息/职业/技能/卡）│
│                         │
├─────────────────────────┤
│  [上一步]    [下一步]     │  ← 底部固定按钮栏
└─────────────────────────┘
```
- 底部固定按钮栏，始终可见
- 每一步顶部有进度指示器（圆点+文字）
- 内容区域可上下滚动

### 1.2 属性掷骰模块

**属性项：**
| 属性 | 掷骰公式 | 显示 |
|------|----------|------|
| STR / CON / DEX / APP / POW | 3D6 × 5 | 掷骰值 + 计算结果 |
| SIZ / EDU / INT | (2D6+6) × 5 | 掷骰值 + 计算结果 |
| 幸运 | 3D6 × 5 | 可选重掷 |

**交互设计：**
- **一键全部掷骰**按钮（大按钮，屏幕中央偏下）
- 点按后依次掷骰动画（每个骰子快速闪烁，最后定格）
- 每个属性右侧有 **🔁 重掷** 小按钮（仅重掷单个属性）
- 全部掷骰后可选择 **全部重掷**（确认弹窗）
- 显示掷骰历史（本角色最多保留 3 次记录）

**本地存储：** 掷骰结果存入 `wx.setStorageSync('coc7_dice_<角色ID>', data)`

### 1.3 基础信息填写

**字段列表：**
- 姓名（文本输入，弹出键盘）
- 玩家（文本输入，弹出键盘）
- 年龄（数字选择器，18-90，默认25）
- 性别（picker 选择：男/女/其他）
- 住地（文本输入）
- 故乡（文本输入）
- 时代背景（picker 选择：1920s / 现代 / 维多利亚 / 自定义）

**手机适配：**
- "姓名"等字段用 `input` + 弹出系统键盘
- 年龄用 `picker` 滚轮选择，避免手输
- 时代选择用底部弹出 picker

**年龄补正（自动计算）：**
| 年龄 | 属性修正 |
|------|----------|
| 15-19 | STR/SIZ 减 1，EDU 减 1，DEX/APP 加 1 |
| 20-39 | 无修正 |
| 40-49 | STR/CON/DEX 减 1，EDU/INT 加 1 |
| 50-59 | STR/CON/DEX 减 2，EDU/INT 加 2 |
| 60-69 | STR/CON/DEX 减 3，EDU/INT 加 3 |
| 70-79 | STR/CON/DEX 减 4，EDU/INT 加 4 |
| 80+ | STR/CON/DEX 减 5，EDU/INT 加 5 |

### 1.4 职业选择系统

**交互设计：**
- 搜索框 + 列表滚动（参考现有 tes-botse-skills 的搜索模式）
- 职业卡片展示：职业名 + 信用评级范围 + 核心技能标签
- 选中后卡片高亮，底部显示详情
- 支持"自定义职业"

**内置 30+ 职业数据（部分）：**
```
会计师、杂技演员、演员（戏剧/电影）、事务所侦探、
精神病医生、动物训练师、文物学家、古董商、考古学家、
建筑师、艺术家、运动员、作家、酒保、拳击手……
```

**选中职业后自动填充：**
- 信用评级范围（CR 范围）
- 职业属性公式（如 `EDU×4`、`EDU×2+APP×2`、`STR×2+DEX×2` 等）
  → 自动计算职业技能点
- 本职技能列表（自动标记 ★/☆/☯/※）
- 推荐关系人

### 1.5 衍生属性自动计算

| 衍生属性 | 公式 | 备注 |
|----------|------|------|
| HP | (CON+SIZ) / 10 向下取整 | |
| SAN | = POW | |
| MP | POW / 5 | |
| DB / Build | 查 STR+SIZ 对照表 | 内置查表逻辑 |
| MOV | 默认 8，年龄/护甲修正 | |
| 重伤值 | HP / 2 | |
| 属性加值 | 属性值 / 10 向下取整 | 半值 |

**手机适配：** 计算结果实时显示，无需手动操作。数据变化即自动重算。

### 1.6 技能分配系统

**技能分类：**
- **职业技能**：由职业决定的本职技能列表
- **兴趣技能**：所有技能均可选

**技能点计算：**
- 职业技能点 = 职业属性公式结果（如 `EDU×4 = 65×4 = 260`）
- 兴趣技能点 = `INT × 2`

**交互设计：**
```
┌─────────────────────────────┐
│ 职业技能点: 260/260  ███████ │  ← 进度条
│ 兴趣技能点: 120/120  ███████ │
├─────────────────────────────┤
│ 会计        50%    [+][-]  │  ← 本职技能(★标记)
│ 人类学      01%    [+][-]  │  ← 不可增减
│ 估价        05%    [+][-]  │  ← 本职技能(★标记)
│ ⋮                          │
│ 拳击        20%    [+][-]  │  ← 兴趣技能
└─────────────────────────────┘
```

**手机关键设计：**
- `[+]` `[-]` 按钮 ≥ 44×44pt，手指好点
- 长按 `[+]` 持续累加（按住不放连续加）
- 本职技能在职业选定后自动**★高亮**
- 技能基础值自动显示（如"会计"基础 05%）
- 每项技能显示：`基础值 + 已分配 = 最终值`
- 提示分配完成度：进度条颜色变化（绿→黄→红）

**符号系统（从 Excel 继承）：**
| 符号 | 含义 |
|------|------|
| ★ | 本职技能 |
| ☆ | 可选本职（多项选一） |
| ☯ | 二选一 |
| ※ | 跨职业 |
| ⊙ | 任意特长 |

### 1.7 角色卡预览

**布局（手机竖屏）：**
```
┌─────────────────────┐
│ 🔍 调查员名称         │
│ 职业 | 时代 | 年龄     │
├─────────────────────┤
│ STR: 60  DEX: 70    │  ← 属性两列排列
│ CON: 50  APP: 65    │
│ POW: 55  SIZ: 55    │
│ INT: 75  EDU: 70    │  ← EDU特殊标记
├─────────────────────┤
│ HP: 11  SAN: 55     │
```

**本地存储：** 保存到 `wx.setStorageSync('coc7_characters', [])` 角色列表数组

## 🟡 第二阶段：装备与战斗系统

> **目标：能添加武器、防具，并查看战斗属性**

### 2.1 武器库引用

**内置数据：** 60+ 武器（近战/远程/火器）
**交互：**
- picker 选择武器分类（近战/手枪/步枪/冲锋枪/散弹枪…）
- 列表滚动选择武器
- 选中后显示详细属性面板

**每条武器数据含：**
```
名称 | 伤害 | 基础值 | 贯穿 | 射程 | 装弹量 | 故障值 | 时代
```

**手机适配：** 列表项紧凑，关键数据（伤害、贯穿）用标签热区显示

### 2.2 武器配备到角色

- 从武器库选择后添加到角色武器栏
- 自动结算 DB 加成 → 最终伤害
- 可装备多把武器（战斗中切换）
- 显示当前熟练技能（如"拳击"技能挂钩徒手攻击）

### 2.3 防具系统

**内置数据：** 12+ 防具类型

| 防具 | 护甲值 | 覆盖部位 | MOV惩罚 |
|------|--------|----------|---------|
| 皮夹克 | 1 | 躯干、手臂 | 0 |
| 防弹背心IIIA | 2 | 躯干 | 0 |
| 防弹背心III | 3 | 躯干 | -1 |
| 摩托头盔 | 2 | 头部 | 0 |
| ⋮ | ⋮ | ⋮ | ⋮ |

**手机适配：** 防具卡片展示，左侧护甲值大字突出，右侧部位徽章

### 2.4 资产与消费水平

**基于信用评级自动换算：**

| 信用评级 | 现金 | 资产 | 消费水平 |

| 0 | $25 | $50 | 贫困 |
| 9 | $45 | $100 | 贫困 |
| 10-49 | $2D10×10 | $2D10×100 | 温饱 |
| 50-89 | $2D10×100 | $2D10×1000 | 中等 |
| 90+ | $5D10×1000 | 5D10×10000 | 富裕 |

**时代换算：** 1920s / 现代 / 维多利亚 自动切换
